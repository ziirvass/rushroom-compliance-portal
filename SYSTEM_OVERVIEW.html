1. System Overview

The Rushroom Compliance Portal is a web-based compliance documentation system that manages the lifecycle of compliance documents, standards, and regulatory requirements for an LED system-furniture product.

Core Concepts

Templates: Reference document types that serve as starting points for operational documents
As Operates: Actual operational compliance documents that reflect current business practice
Standards: Regulatory requirements (e.g., EN 60598-1, CE marking directives)
Versions: Immutable historical records; no documents are ever deleted, only versioned
Provenance: Explicit tracking of which template versions and standard versions were used to create each document version
AI-Assisted Drafting: Claude API generates document updates based on current version + standards + context

7. Key Features & Guarantees

Immutable Versioning

  • Never delete: All documents and standards are version-controlled. No row is ever deleted.
  • Full history: Each version row has created_at, uploaded_by, and notes for audit trails.
  • Provenance: Each document_version tracks its source: source_document_version_id and source_standard_version_ids[].

AI-Assisted Drafting

  • Long standard handling: Standards >20 pages are summarized by Claude before drafting to avoid token overflow.
  • Multi-source drafting: AI can read from current document + multiple standard versions + user context simultaneously.
  • Structured output: Claude returns JSON with proposed changes, draft text, version hint, and file name suggestion.
  • Human-in-the-loop: User approves individual changes via checkboxes before publishing.

Standards Register

  • Version tracking: Each standard has multiple versions (2015, 2015+A1, 2015+A1:2022, etc.) with effective dates.
  • Compliance audit: AI can compare all operational documents against the entire standards register in one scan.
  • Deviation findings: Results stored with severity levels (Critical, High, Medium, Low, Info) and actionable recommendations.

Google Docs Integration

  • Seamless editing: After AI draft, user clicks "Edit in Google Docs" to polish the document.
  • Full formatting: Users can add tables, images, comments, track changes, and collaborate.
  • Fetch back: Portal retrieves the edited content and updates the draft before publishing.

Role-Based Access Control

  • Rushroom: Full edit of documents, standards, action plan, and AI controls.
  • Supplier: Read-only view of assigned steps; can upload files and see status.
  • Reviewer: Read-only access to specific documents marked for review.
  • Installer: Read-only access to installation/deployment guides.

11. Level 2 Architecture: Structured Clause-Level Interpretations New in Level 2

The system now supports a Level 2 data architecture that decomposes compliance documents into atomic, machine-queryable records. This enables:

  • Direct queries like "what is our interpretation of EN 60598-1 clause 4.11?" without parsing document prose
  • Automated DPP (Digital Product Passport) export in JSON-LD format for EU ESPR compliance
  • Faster, cheaper deviation scanning (using structured data before AI inference)
  • Audit trail with human review tracking at the atomic level

Updated Deviation Scan (Phase 3 Complete)

The runDeviationScan action now uses a two-tier approach:

  1. Structured tier: Query as_operates_interpretations for rows with compliance_status = 'deviation' or 'pending'. These become instant findings with source = 'structured'. No LLM needed; faster and cheaper.
  2. AI fallback: For documents without interpretations or for legacy analysis, call Claude API as before. Mark findings with source = 'ai_inference'.

Result: Scans that have structured interpretation data run ~50x faster and cost ~50x less than full AI scans.

DPP Export & ESPR Compliance (Phase 4 Complete)

The system is now DPP-ready for EU Digital Product Passport requirements (furniture mandatory from 2027). The exportProductPassport action returns:

JSON-LD format: Conforms to schema.org/Product + ESPR custom extensions, ready for GS1 Digital Link QR codes
Compliance linkage: Passports reference specific interpretation records showing which clauses are met and which have deviations
Sustainability data: Stores CO2, materials, recyclability — pulled from interpretation metadata
PDF embedding: pdf-data format can be embedded in PDF metadata for offline distribution

12. EU Directive Relationship Analyser (CELLAR) New module

A dedicated module that maps and visualises how EU directives/regulations relate to each other — at company level (which apply to Rushroom AB as a business) and at product level (which apply to a specific product passport, e.g. the LED wardrobe system). Relationship data is drawn from the EU Publications Office CELLAR repository (public SPARQL + REST, no auth) and, on demand, from Claude AI inference.

Data sources

  • SPARQL endpoint: https://publications.europa.eu/webapi/rdf/sparql — queried for cdm:resource_legal_based_on, _implements, _amends, _repeals and cdm:work_cites_work triples (mapped to relation types requires/implements/amends/supersedes/references, confidence 1.0).
  • REST content API: https://publications.europa.eu/resource/celex/{CELEX} — Akoma Ntoso / XHTML parsed for explicit <ref> cross-references, attributed to the nearest article (akn_ref_element, confidence 1.0).
  • AI inference (on demand only): Claude reads article text and surfaces implicit relations (ai_inferred, confidence 0.0–1.0, always shown in the UI). Never runs automatically — cost control.

New tables (supabase/schema.sql)

  • eu_directives — registry (celex_number, short_name, official_title, directive_type, status, in_force_date, scope_description, eur_lex_url, applies_to_company). Seeded with the 7 directives relevant to the LED wardrobe system (LVD, EMC, RoHS, WEEE, Machinery, Batteries, ESPR).
  • directive_relations — source/target directive links (relation_type, source/target clause refs, source = cellar_sparql | akn_ref_element | ai_inferred, confidence, verified). target_celex is kept even when the target isn't in the registry yet (a gap).
  • product_directive_applicability — directive applicability per product passport (applicable | not_applicable | partial | under_review + rationale).
  • cellar_cache — raw CELLAR results by (celex_number, query_type); re-fetched only if older than 7 days.

All four tables have RLS enabled (deny-all); access is only via the edge function's service role.

CELLAR service (portal-api/cellar-service.ts)

createCellarService(db, anthropicKey) exposes fetchDirectiveMetadata, fetchDirectiveRelations, fetchDirectiveFullText and extractImplicitRelations. All CELLAR access is rate-limited (max 5 concurrent), retried with exponential backoff on 429/503, bounded by a 60s timeout, and 7-day cached. A failure degrades gracefully (null/[]) so the portal keeps working when CELLAR is unreachable.

New API actions (edge function)

  • listDirectives — registry (+ applicability for a passport).
  • addDirective {celexNumber, shortName, appliesToCompany} — fetches CELLAR metadata, upserts.
  • syncDirectiveRelations {directiveId} — SPARQL + AKN <ref> relations upserted; returns {relationsFound, inserted, skipped}.
  • inferDirectiveRelations {directiveId} — on-demand AI inference (never automatic).
  • analyseComplianceGraph {scope, passportId?} — returns {nodes[{id,shortName,status,complianceCoverage}], edges[{source,target,relationType,clauses}], gaps[{celex,reason}]}.
  • setDirectiveApplicability {passportId, directiveId, status, rationale}.
  • getComplianceCoverage {directiveId} — clause-level coverage (total/covered/pct/pending/deviation), tied to standards whose domain matches the directive short-name.
  • generateComplianceNarrative {scope, passportId?, language: sv|en} — Claude writes a human-readable narrative from the graph, for a DoC / passport.

Visualisation (Directive Graph tab, Rushroom only)

A D3.js 7.9.0 force-directed graph (CDN). Nodes = directives, sized by number of applicable clauses, coloured by compliance coverage (green >80%, amber 40–80%, red <40%, grey not assessed). Edges styled by relation type (solid requires/implements, dashed references/amends, dotted AI-inferred) with confidence-scaled width. Click a node → sidebar (title, scope, coverage bar, related directives, EUR-Lex + standards-register links); click an edge → relation type, clauses, description, confidence, source. A Company / Product toggle (product selector in product mode), a gaps table ("Add to portal" per referenced-but-missing directive), and an AI compliance narrative export complete the view.
